Learning Git

This article is based solely and exclusively on my knownledge of Git!

What Is Git?

Git is a versioning system that saves the changes of a repository in a separated one through commits which work as snapshots of a piece of code.

If you want more info about git see the official website.

Why Git?

Because every programmer use it, and you must use it too, is like a standard in code versioning and also because Git is good and Mr. Torvalds is clever than your entire family tree.

How Does Git Work?

As far as I understood git works in a very peculiar way compared to other versioning system, instead of saving files to save the changes of a code and turning development a mess, git doesn't save the changes on the files but in commits which are similar to screenshots you often take on your PC.

Git creates a special repository called .git in the directory you selected to 'host' that repository. This repo saves the changes you've made through commands you give in the Git BASH or via terminal.

Starting Git

Firstly you have to 'declare' to git who you are, for this you open the BASH and write the command $ git config --global user.name "Your Name", after this write the other essencial command to git identify you $ git config --global user.email yourEmail@example.com. Git doesn't really "identifies" you, this is bullshit, but the greatest thing about learning and understanding git is cooperate with others and in order to do this you'll need to give these trivial personal info to github or you won't be able to use it properly and share your code with the world.

Thereafter you can start the git really. Write $ git init to initialize the .git repo on your folder.

Some Famous Concepts Before Starting To Really Use it

Commit
A commit is like a save button, it "prints" the code that was added by $ git add -A on the branch you are currently working.
Branch
Branch is a sort of "photo gallery" of your code, is where your changes are saved on the repository. It is called "branch" because it really works like one. Imagine you want to change a piece of code but you need to keep the code the same. A paradox right? That is one of the situations you can create a new branch of the current branch you are working on. The branch will take everything from the branch that created it and you will be able to add code in this new branch without changing anything on the previous branch
Repo
Kind of obvious but a repo is an abbreviation for repository and a repository is a place where you store things, in this case files and folders.
Gitignore
This is a file you can create on your master branch that will ignore untracked files you indicate there. See the docs about gitignore.
Push
Push is actually a git command ($ git push) which enables you to... push a local repository (one in your PC) to a remote one on GitHub.
Pull
"The oposite", you may have thought, and you are right, when you pull a repository it means that you are bringing what is into it to your local repository. Is also a git command ($ git pull).

Really Using it

Now to really use it you should open your repository create a file (if you didn't do it yet) and write something, to identify what you did as a change you must write $ git add -A in BASH and to "send it" to the branch you are (by) you must write $ git commit -m "some message explaining what you did", so simple as that.

To create a new branch write: $ git branch branchName. To see it write: $ git branch and the branch you are currently in will apear brither and with a * before it. To move right to the branch you created write: $ git checkout branchYouWant.

Main Commands On Local Repositories

The git main commands after you started a git repo are the following:

Command Description
git add -A Adds all the changes made in the repository to the branch you're working in.
git commit -m "some message" Commits the changes you've made with a message explaining what was done
git commit -am "some message" This is similar to the previous, the difference is the -a before the m (message). It will commit all the changes made in the files that were already tracked without the need to add them with git add -A
git status Shows the status of the changes in the files, shows the changes that weren't added to the branch, those which were added but not committed yet and also will show a message if there's nothing to add
git log Shows all the commits of a branch, the author of them, the date when they were committed and the message that was left for each one of them.
git branch Shows all the branches you've created and marks with "*"" the one you are.
git branch -m currentBranchName newBranchName Renames the current branch to a new one.
git branch -D branchName Deletes the indicated branch
git checkout branchName Goes to the indicated branch
git reset --soft The most recent commit fades but the code does not change and all the changes were still added, but not commited.
git reset --mixed Similar to --soft but the changes were not added.
git reset --hard Ignores everything that was done on the actual commit and goes back to the previous one.
git diff Shows all the changes of all the files.
git diff somefile.ext Shows all the changes on the indicated file.
git revert --no-edit commitCode Creates a new commit without the changes made on the previous one. You have to write git log before and copy the commit code you want to revert to.
rm -rf .git Removes the .git repository of the folder you are currently in.

Remote repos

But the coolest thing to do with git is to create a remote repository for others to access and see your work and in order to do this you'll need to create a remote repository in the most famous git repos collection of the universe, the GitHub.

The remote will work as a... remote, another repository of the same in your machine. And everything done in the remote will not go to the local unless you give a pull, and everything done in the local will not go to the remote unless you give a push.

Commands to remote

Commands Description
git remote add remoteName linkOfTheRemote This command will create a new remote repository in the indicated link
